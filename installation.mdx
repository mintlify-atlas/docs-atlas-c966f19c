## Prerequisites

Before you begin, ensure you have the following:

<AccordionGroup>
  <Accordion title="TypeScript Compiler" icon="code">
    **Version:** TypeScript 4.0 or higher
    
    ```bash
    # Install globally
    npm install -g typescript
    
    # Verify installation
    tsc --version
    ```
    
    The project uses TypeScript with strict type checking enabled.
  </Accordion>
  
  <Accordion title="Modern Web Browser" icon="browser">
    **Required features:**
    - HTML5 Canvas support
    - ES2015 module support
    - requestAnimationFrame API
    
    **Recommended browsers:**
    - Chrome 61+
    - Firefox 60+
    - Safari 11+
    - Edge 79+
  </Accordion>
  
  <Accordion title="Text Editor (Optional)" icon="pen">
    Any code editor works, but these provide excellent TypeScript support:
    - Visual Studio Code (recommended)
    - WebStorm
    - Sublime Text with TypeScript plugin
  </Accordion>
</AccordionGroup>

## Step-by-Step Installation

<Steps>
  <Step title="Download the source code">
    Clone the repository or download the source files:
    
    ```bash
    git clone https://github.com/Ghostmaysam1/gravity-sim.git
    cd gravity-sim
    ```
  </Step>
  
  <Step title="Understand the TypeScript configuration">
    The project includes a `tsconfig.json` with the following key settings:
    
    ```json
    {
      "compilerOptions": {
        "target": "es2015",
        "module": "es2015",
        "rootDir": "./src",
        "outDir": "./dist",
        "moduleResolution": "bundler",
        "strict": true,
        "esModuleInterop": true,
        "forceConsistentCasingInFileNames": true,
        "skipLibCheck": true
      }
    }
    ```
    
    <Info>
      The `"moduleResolution": "bundler"` setting allows the TypeScript compiler to work with ES modules that will be loaded directly in the browser.
    </Info>
  </Step>
  
  <Step title="Compile the TypeScript source">
    Run the TypeScript compiler from the project root:
    
    ```bash
    tsc
    ```
    
    This generates JavaScript files in the `dist/` directory:
    - `dist/main.js`
    - `dist/Body/Body.js`
    - `dist/drawing/TinyCanvas.js`
    - `dist/types/index.js`
    - `dist/shared/shared.js`
  </Step>
  
  <Step title="Open the HTML file">
    The `index.html` file loads the compiled code as an ES module:
    
    ```html
    <!DOCTYPE html>
    <html lang="en">
    <head>
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <style>
            html, body {
                height: 100vh;
                display: flex;
                justify-content: center;
                align-items: center;
            }
        </style>
        <title>Gravity simulation</title>
    </head>
    <body>
        <canvas id="canvas"></canvas>
        <script type="module" src="./dist/main.js"></script>
    </body>
    </html>
    ```
    
    Open this file in your browser - no web server required!
  </Step>
</Steps>

## Project Structure Overview

Here's how the codebase is organized:

```
gravity-sim/
├── src/
│   ├── main.ts                 # Entry point - creates bodies and starts the loop
│   ├── Body/
│   │   └── Body.ts            # Body class with physics calculations
│   ├── drawing/
│   │   └── TinyCanvas.ts      # Canvas rendering and animation loop
│   ├── shared/
│   │   └── shared.ts          # Constants (gravitational constant G)
│   └── types/
│       └── index.ts           # TypeScript type definitions
├── dist/                       # Compiled JavaScript (generated by tsc)
├── index.html                  # Entry HTML file
└── tsconfig.json              # TypeScript compiler configuration
```

<Note>
  The `dist/` directory is created automatically when you run `tsc`. You don't need to create it manually.
</Note>

## Running the Simulation

### Local File Method

Simply open `index.html` directly in your browser:

```bash
# macOS
open index.html

# Linux
xdg-open index.html

# Windows
start index.html
```

### Development Server Method

For a better development experience with auto-reload, use a local server:

<CodeGroup>
```bash Python
# Python 3
python -m http.server 8000

# Then open http://localhost:8000
```

```bash Node.js
# Install http-server globally
npm install -g http-server

# Run server
http-server -p 8000

# Then open http://localhost:8000
```

```bash PHP
php -S localhost:8000

# Then open http://localhost:8000
```
</CodeGroup>

<Tip>
  Using a development server prevents CORS issues if you modify the code to load external resources.
</Tip>

## Troubleshooting Common Issues

<AccordionGroup>
  <Accordion title="Error: Cannot find module" icon="triangle-exclamation">
    **Symptom:** Browser console shows "Failed to load module script"
    
    **Solution:**
    - Ensure you've run `tsc` to compile TypeScript to JavaScript
    - Check that the `dist/` directory exists and contains `.js` files
    - Verify that imports in your `.ts` files use `.js` extensions (e.g., `'./Body/Body.js'`)
    
    ```typescript
    // Correct
    import Body from './Body/Body.js';
    
    // Incorrect
    import Body from './Body/Body';
    ```
  </Accordion>
  
  <Accordion title="Blank canvas or nothing appears" icon="eye-slash">
    **Symptom:** The page loads but shows a blank screen
    
    **Solution:**
    - Open browser DevTools (F12) and check the Console for errors
    - Verify that the canvas element exists: `document.getElementById('canvas')`
    - Check that compiled JavaScript files are being loaded in the Network tab
    - Ensure your browser supports ES2015 modules
  </Accordion>
  
  <Accordion title="TypeScript compilation errors" icon="code">
    **Symptom:** `tsc` command fails with type errors
    
    **Solution:**
    - Ensure you're using TypeScript 4.0 or higher: `tsc --version`
    - Check that `tsconfig.json` exists in the project root
    - If you've modified the code, ensure all types are correct
    - Run `tsc --noEmit` to check for errors without generating files
  </Accordion>
  
  <Accordion title="Bodies don't move or physics seems broken" icon="atom">
    **Symptom:** Bodies appear but don't interact gravitationally
    
    **Solution:**
    - Verify that at least one body is NOT static (`isStatic: false` or undefined)
    - Check that bodies have appropriate mass values (not zero)
    - Ensure the animation loop is running (check FPS counter in top-right)
    - Look for JavaScript errors in the browser console
  </Accordion>
  
  <Accordion title="Poor performance or low FPS" icon="gauge-low">
    **Symptom:** Simulation runs slowly with many bodies
    
    **Solution:**
    - The simulation has O(n²) complexity - performance degrades with many bodies
    - Reduce the number of bodies in the simulation
    - Consider implementing spatial partitioning (quadtree) for large simulations
    - Check browser hardware acceleration is enabled
    - Close other resource-intensive applications
  </Accordion>
</AccordionGroup>
